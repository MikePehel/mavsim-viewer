Underwater View Modes

Toggle underwater mode with U. Each view mode gets a submerged variant. The water surface at Y=0 becomes a caustic-lit ceiling. The vehicle descends into negative Y space. No seafloor — just open water fading to fog.

Toggle: KEY_U toggles is_underwater on the scene. Eventually this will auto-activate when mav_type == 12 (MAV_TYPE_SUBMARINE) and we have a proper ROV model. Until then, manual toggle lets us develop and test the rendering with any vehicle. The V key still cycles view modes — each renders its underwater variant when the flag is set.

Spatial Model

The fundamental shift: the ground plane becomes the water surface, and the vehicle operates below it.

Y=0 is the Water Surface

The existing ground plane mesh stays at Y=0 but renders as a semi-transparent water surface with animated caustic patterns instead of grid lines. Double-sided — visible from above and below. This is the vehicle's spatial reference: it looks up and sees the shimmering surface.

rlDisableBackfaceCulling() on surface mesh
Caustics replace grid lines in shader
Grid uniforms → caustic uniforms

Negative Y = Depth

Vehicle position is negative Y. Depth 22m = position Y=-22. This matches PX4's convention: altitude is relative to home (surface), going down is negative. The HUD shows depth as a positive number with a "DEPTH" label instead of "ALT".

depth_display = -vehicle.position.y
HUD label: "ALT" → "DEPTH"
Vert speed arrow meaning flipped

No Seafloor

There is no ground plane below the vehicle. Looking down shows open water fading into fog. The fog IS the ocean — depth-based visibility limits create the sense of a water column without needing geometry. This is accurate to open-water diving.

Fog range: 60-200m (mode dependent)
Fog color: mode's water color
Below = void, above = caustic surface

Color Attenuation with Depth

Water absorbs red light first, then green. Deeper = bluer. Trail colors and vehicle tint shift toward blue-green based on the vehicle's Y position. The HUD is exempt — instruments don't attenuate.

red *= exp(position.y * 0.08)
green *= exp(position.y * 0.02)
blue unaffected

Caustic Surface (Replaces Grid)

The grid shader renders procedural caustics instead of grid lines when is_underwater is set. The caustic pattern provides the same spatial grounding — you can perceive relative motion against it — but reads as light refracting through surface waves.

Caustic Surface — Looking Up from -15m

Caustics replace the grid — same shader, different math:

// In grid.fs, when is_underwater:
// Instead of fract()-based grid lines, compute caustic pattern

float caustic(vec2 uv, float time) {
    vec2 uv1 = uv * 8.0 + vec2(time * 0.3, time * 0.2);
    vec2 uv2 = uv * 6.0 + vec2(-time * 0.2, time * 0.35);

    float c1 = sin(uv1.x + sin(uv1.y * 0.8))
             * sin(uv1.y + sin(uv1.x * 0.6));
    float c2 = sin(uv2.x + sin(uv2.y * 0.7))
             * sin(uv2.y + sin(uv2.x * 0.9));

    float c = (c1 + c2) * 0.5;
    return clamp(c * c, 0.0, 1.0);
}

// In main():
if (is_underwater) {
    float c = caustic(worldPos.xz * 0.01, time);
    fragColor.rgb = groundFill + causticColor.rgb * c * 0.3;
    // No grid lines, no axis lines, no major/minor distinction
    // Just the organic caustic web
} else {
    // existing grid line logic
}

New uniforms needed: is_underwater (bool/int), time (float). Caustic intensity fades with camera depth — bright at -2m, gone by -80m. Same shader file, same draw call.

Edge Rings (Replace God Rays)

Instead of volumetric light shafts, randomly-sized circle rings drift at the edges of the viewport. They evoke light refracting through the water column — distant, ambient, not directional. Subtle and organic.

Edge Rings — 2D Screen-Space Overlay

Edge rings are a 2D screen-space overlay, drawn after EndMode3D():

// ~8-12 ring slots, each with random state
typedef struct {
    float x, y;         // screen position (spawns at edge)
    float radius;       // 20-80px (randomly assigned on spawn)
    float dx, dy;       // drift velocity (toward center, slow)
    float life;         // 0→1, fades in then out
    float life_max;     // 4-8 seconds
    float thickness;    // 1.0-2.0px
} edge_ring_t;

// On spawn: pick random edge (top/bottom/left/right)
// Position starts partially off-screen
// Drift: 5-15px/sec inward + slight lateral wander
// Alpha: fade in over 15%, hold, fade out over 20%
// Color: mode accent at 0.05-0.12 alpha
// On death: respawn at new random edge position

// Draw with DrawCircleLines() or DrawRing() in 2D overlay
DrawRing(center, ring->radius - 0.5f, ring->radius + 0.5f,
         0, 360, 36, Fade(accent, alpha));

~40 lines of C. No GPU cost beyond a handful of DrawRing calls. Stays at the periphery — never obscures the vehicle or HUD. Feels ambient, not distracting.

Key Differences from Other Modes

No Surface Reference

  • At 200m+ depth, the caustic surface is invisible — too far and too dark
  • No caustic shader runs — surface plane not rendered
  • Spatial reference comes from bioluminescent particles and the trail only
  • Fog range extremely short: 20m → 60m
  • This is intentionally disorienting — deep sea IS disorienting

Vehicle Headlight

  • Radial gradient centered on vehicle's forward direction
  • Only light source in the scene (besides bioluminescence)
  • Illuminates particles and nearby water
  • Could be a simple additive-blend billboard quad
  • Consistent with air Rez: the vehicle stands out against darkness

Ortho Panels

Side View — The Best One

  • Horizontal water surface line at Y=0 in the mode's accent color (2px)
  • Above the line: sky color (lighter)
  • Below the line: water color gradient (darker with depth)
  • Vehicle and trail visible below the line
  • Depth markings on Y axis replace altitude markings
  • Caustic shimmer along the surface line (subtle animated pattern)
  • This view immediately communicates "underwater" — the split is iconic

Top-Down View

  • Looking straight through the water surface
  • Background: mode's water color (uniform, no gradient)
  • No caustics visible (looking perpendicular to surface)
  • Edge rings still render (screen-space overlay)
  • Trail and vehicle clearly visible
  • Functionally identical to air mode — just recolored
