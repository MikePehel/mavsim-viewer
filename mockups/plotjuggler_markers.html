CSV Output Format

The exported CSV is designed for direct import into PlotJuggler as a secondary data source. The marker column acts as a pulse signal — it's 1.0 at each marked timestamp and absent elsewhere, so PlotJuggler draws it as vertical spikes on the timeline.

# MAVSim flight markers — exported 2026-03-10T14:23:07
# Load in PlotJuggler: File → Load Data, select this CSV
# Drag "marker" column onto any plot as vertical reference lines
timestamp_s,marker,label,altitude_m,ground_speed_ms,roll_deg,pitch_deg,heading_deg
14.203,1.0,marker_1,12.4,8.2,3.1,-2.5,045
42.871,1.0,anomaly,48.7,15.1,-22.4,5.8,178
61.445,1.0,roll_osc,52.1,14.3,38.9,-1.2,220
94.112,1.0,flare,8.2,6.4,1.8,8.5,045

Why This Format?

timestamp_s (seconds)

  • PlotJuggler expects seconds as the time axis
  • Converted from time_usec ÷ 1,000,000
  • Relative to boot time — same clock as ULog
  • Sub-millisecond precision preserved

marker (pulse signal)

  • Always 1.0 — PlotJuggler renders as discrete spikes
  • Drag onto any existing plot as an overlay
  • Vertical lines appear exactly at marker timestamps
  • Works with PlotJuggler's "XY plot" and "TimeSeries" views

Snapshot Telemetry

  • Captures vehicle state at the exact moment of marking
  • Useful for quick reference without scrubbing ULog
  • Each column is independently plottable in PJ

label (string)

  • PlotJuggler shows string columns in its data table
  • Searchable in PJ's sidebar
  • Auto-generated if user doesn't specify

Timestamp Alignment

Clock Synchronization

Both the ULog and our markers use time_usec from the flight controller's boot clock. Since our viewer receives this timestamp directly in HIL_STATE_QUATERNION, the marker timestamps should align with the ULog timeline with no offset — they're the same clock source. For replay mode, timestamps come directly from the ULog itself, so alignment is guaranteed.

Live Flight (UDP)

  • time_usec from HIL_STATE_QUATERNION
  • Same boot-relative clock as ULog on SD card
  • Network latency adds ~1-5ms offset (negligible)
  • Marker timestamp = last received time_usec

Replay Mode (ULog)

  • Timestamps read directly from ULog file
  • Perfect alignment guaranteed — same source
  • Pause + mark works (uses last played timestamp)
  • Seek doesn't affect previously placed markers
